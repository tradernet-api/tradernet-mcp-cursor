# /tn-connect — проверка подключения Tradernet MCP

Выполни по порядку, останавливаясь на первой ошибке и подсказывая причину.

1. **Сервер виден.** Убедись, что MCP-сервер `TN` подключён и инструменты
   доступны. Если список пуст — попроси перезапустить Cursor из терминала после
   `source ~/.config/tn-mcp/credentials.env`.

2. **Рынок (без ключей).** Вызови `reference_get_market_status`.
   Ошибки авторизации тут быть не должно — проверяет сам канал.

3. **Котировки (HMAC).** Вызови `quotes_get` с `{ "tickers": ["AAPL.US"] }`.
   - `HMAC credentials are required` → env не подхвачен, перезапуск Cursor.
   - `Invalid signature provided` → перевыпустить ключи (см. правило `tn-mcp-auth`).

4. **Портфель (HMAC).** Вызови `portfolio_get`. Работает только через HMAC.

5. **(Опционально) SID.** Если нужен SID-канал — вызови `get_sid_info`.
   При `401 Invalid credentials` запусти скилл `tn-mcp-refresh-sid`.

В конце кратко отчитайся: какие шаги прошли, какой канал работает (HMAC / SID),
и что делать, если что-то упало. Числа в ответах могут приходить строками —
это нормально.
